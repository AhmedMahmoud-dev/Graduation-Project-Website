Image & Video Emotion Model

How Emotra analyzes static images and dynamic video files to detect, track, and aggregate facial expressions using high-speed deep neural networks.

The visual emotion engine operates on a unified model system to process both static images and video clips. By utilizing the same core face detection and expression classification networks, Emotra guarantees consistent prediction baselines across both modalities, adding tracking and smoothing steps specifically optimized for video temporal layers.

In our previous iteration, we used RetinaFace, which created significant CPU bottlenecks. By upgrading to OpenCV's built-in **res10 SSD detector**, we execute face detection in pure C++ inside OpenCV. This reduces processing times from 8 seconds down to less than 100ms per frame.

Raw coordinates returned from the SSD detector are expanded dynamically before cropping. Adding **15% horizontal padding** and **25% vertical padding** ensures that the model preserves structural landmarks (such as foreheads and jawlines) which are critical for accurate HSEmotion predictions.

Videos present a massive processing burden on a single thread due to Python's Global Interpreter Lock (GIL). Emotra resolves this by separating frames decoding from inference. It loads independent model instances inside a **ProcessPoolExecutor** worker pool, achieving real multi-core parallelization.

Adaptive Frame Rate (FPS)

Rather than analyzing every single frame (e.g. 30 FPS), we target a lower sampling frequency that dynamically shrinks as the video gets longer. Videos shorter than 5 seconds are sampled at 4.0 FPS, while clips longer than 3 minutes scale down to 0.5 FPS.

Frame-Difference Skipping

To prevent redundant CPU iterations on static clips, we check downsampled frame similarity. If the current frame is more than **98.5% similar** to the last sampled frame, it skips model inference entirely.

When multiple faces are present in a video, individual frames cannot easily group which faces belong to which person. Emotra integrates **DeepSORT** tracking, assigning persistent tracking IDs to face bounding boxes based on velocity and appearance overlaps.

Additionally, face expressions shift rapidly, which creates flickering predictions. We apply a **Bidirectional Exponential Moving Average (biEMA)** smoothing pass. By calculating EMA trends forward and backward in time, we eliminate causal tracking delay and align transitions precisely with the video frames.

The API returns complete bounding box tracks, processing speeds, downscaling metadata, and sorted emotion confidence percentages for every tracked face.

Face Occlusions

If a subject places their hand over their face, wears sunglasses, or turns away from the camera (profile view), landmarks cannot be aligned. The SSD detector may fail or return low confidence scores that trigger fallback neutral predictions.

Extreme Lighting Changes

Severe shadows, dark environments, or light glares hide subtle facial muscle movements, causing false drops in confidence levels. Consistent ambient lighting is recommended for accurate results.

Scale-Down Limitations

All frames larger than 640px are downscaled prior to face detection to maintain rapid execution speeds. Highly detailed, small facial shapes in crowds may be lost during downscaling and skipped.
