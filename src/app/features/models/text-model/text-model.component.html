Text Emotion Model

How Emotra analyzes text to detect emotions — the AI architecture, processing pipeline, and the science behind every prediction.

Emotra's text emotion engine takes any English text — from a single sentence to full paragraphs — and detects the emotional tone behind the words. The model classifies text into 7 distinct emotions with confidence percentages, giving you a detailed breakdown of the emotional landscape.

At the core of Emotra is DistilRoBERTa, a transformer-based natural language processing (NLP) model fine-tuned specifically for emotion detection. It was trained on thousands of labeled text samples across 7 emotion categories by j-hartmann and published on Hugging Face.

Instead of treating your entire text as one block, Emotra splits it into individual sentences. This allows us to detect emotional shifts within a paragraph — for example, a message that starts with joy and ends with sadness.

Sentences are split on period (.), exclamation (!), and question (?) marks. Each sentence is then analyzed independently, and the results are combined.

Emotra uses a keyword-based intensity system to give more influence to sentences that contain highly emotional language. When a sentence contains words like "devastated", "terrified", or "furious", its weight is boosted significantly.

Weight Modifiers
Strong Boost
Fear: "terrified", "panic", "dread" → +1.1
Sadness: "devastated", "grief", "broken" → +1.5
Anger: "furious", "rage", "irate" → +1.2
Disgust: "revolting", "disgusting" → +1.3
Reduction
Relief: "relieved", "calm" → −0.5
Neutral: "normal", "fine", "okay" → −0.7
Minimum weight floor: 0.3

Each sentence is tokenized using the model's tokenizer (up to 512 tokens), passed through the transformer, and the output logits are converted to probabilities via softmax. The result is a list of 7 probability values — one for each emotion — that sum to 1.0.

torch.no_grad() disables gradient computation during inference, reducing memory usage and improving speed — we're not training, just predicting.

Emotra uses a dual-analysis fusion approach. First, each sentence is analyzed independently and weighted by its emotional intensity. Then, the full text is analyzed as one piece to capture the overall context. Finally, both are blended together.

sentence_weightedi + full_texti × 0.8

The full-text analysis carries an 80% weight influence. This balances granular sentence-level detail with holistic paragraph context.

Token Limit (512)

Input longer than 512 tokens is automatically truncated. The remaining text is not analyzed. The API response includes a flag indicating if truncation occurred.

English Only

The model was trained exclusively on English text. Using other languages may produce unreliable or inaccurate results.

No Sarcasm Detection

The model takes text at face value. Sarcastic statements like "Oh great, another meeting" will likely be classified as joy/surprise rather than frustration.

Keyword Bias

The intensity weighting system relies on predefined keyword lists. Text using synonyms not in the list may not receive appropriate weighting boosts.
