Before analysis begins, the pipeline "cleans" the audio. We use adaptive noise reduction to subtract static hiss and environmental noise, followed by peak-normalization. The system also flags digital clipping, which could lead to "angry" false positives.

The pipeline segments raw audio into 2.0s windows with a 1.0s overlap. Instead of treating every chunk equally, we apply Voice Activity Detection (VAD) logic to weight "speech-clear" segments higher than whispers or background hums.

We use emotion2vec+ base (from Alibaba FunASR), a model trained on massive amounts of unlabeled audio using self-supervised learning. Unlike HuBERT, it provides native high-fidelity detection for 7 distinct emotion classes without synthetic mapping.

Traditional real-time models suffer from a "causal lag" where the smoothed emotion trail is always 1-2 seconds behind reality. Emotra v5.0 uses Bidirectional EMA: it runs a forward pass and a backward pass simultaneously and averages them. This places emotion transitions exactly where they happen in time.

In v1.0, audio and text were always 50/50. In v5.0, we use Entropy-Aware weighting. We calculate the Shannon entropy (uncertainty) for both channels. If the audio is garbled but the words are clear, text gets higher weight. If the transcription is shaky but the tone is clearly "angry", audio dominates.

Confidence Gating

Segments where the model is highly uncertain (max prob < 0.25) are severely downweighted (×0.15) to prevent trash-in/trash-out errors.

Adaptive Entropy

The system automatically detects when one modality is failing (asymptotic entropy) and shifts the entire model's trust to the surviving channel.
