---

WHAT THE COMPARE PAGE DOES:

It allows the user to select two past analyses of the same type (text vs text OR audio vs audio) from their history and see a deep side-by-side emotional comparison between them. This is called an "Emotional Diff" — like a code diff but for emotions.

---

LOCALSTORAGE DATA:

The history page stores analyses in localStorage. Each analysis type has its own key. Each entry contains the full API response JSON — not a summary, the complete response including all sentences, all probabilities, timeline data, model info, timestamps, everything.

Before writing any code, read the existing history page and localStorage service to understand exactly how entries are stored and retrieved. Use the same retrieval logic — do not create a new storage system.

Each entry has its own unique ID. Entries are either text analyses or audio analyses stored under separate keys.

---

ROUTE:

/compare → AppLayout → CompareComponent [AuthGuard]

Add this route to app.routes.ts as a child of the AppLayout route.

PAGE STRUCTURE:

The page has 6 sections in this order:

SECTION 1 — SELECTION PANEL
SECTION 2 — DOMINANT EMOTION HERO ROW
SECTION 3 — TIMELINE OVERLAY CHART
SECTION 4 — EMOTION DISTRIBUTION COMPARISON
SECTION 5 — SENTENCE BY SENTENCE DIFF
SECTION 6 — STATS SUMMARY TABLE

All sections below Section 1 are hidden until both analyses are selected. When both are selected, all sections animate in smoothly (fade in, stagger).

---

SECTION 1 — SELECTION PANEL

Two side-by-side panels. Each panel is a large clickable card that acts as a selector.

Before selection:
- Left card: dashed border, icon in center, label "Select Analysis A"
- Right card: dashed border, icon in center, label "Select Analysis B"
- A type selector above both cards: two toggle buttons "Text" and "Audio" — selecting a type filters which analyses appear in the picker modal
- Both cards show the same type — you cannot compare text vs audio

When a card is clicked, open a PICKER MODAL:
- Modal shows all history entries of the selected type
- Each entry in the modal is a card showing:
  - Date and time (formatted nicely)
  - First 80 characters of the input text/audio name
  - Dominant emotion badge (colored with emotion color)
  - Confidence percentage
  - Sentence/segment count
- Entries are sorted by date, newest first
- User clicks an entry to select it — modal closes, card updates to show the selected analysis summary
- If the same entry is already selected in the other panel, do not allow selecting it again — show it grayed out with a "Already selected" label

After selection, each panel card shows:
- Date and time
- First 80 characters of input
- Dominant emotion (large, colored)
- Confidence percentage
- A "Change" button to reopen the picker

A "Compare" button appears between the two cards once both are selected (or comparison loads automatically — your choice, automatic is better UX).

If no history entries exist for the selected type: show an empty state inside the modal with a message and a link to go analyze something first.

SECTION 2 — DOMINANT EMOTION HERO ROW

CENTER COLUMN — The Difference:

- A sentiment shift label:
  - If A was negative and B is positive: "Improved" in green with up arrow
  - If A was positive and B is negative: "Worsened" in red with down arrow
  - If similar category: "Similar" in neutral color
- Confidence delta: show the difference in confidence percent (+12% or -8%)

RIGHT COLUMN — Analysis B:
- Same as left column

Subtitle: "Both emotional journeys on one chart"

RESPONSIVENESS REQUIREMENTS (MANDATORY):

SCREEN SIZES:

Large desktop (1440px+):

- Selection panel: two cards side by side

- Distribution charts: two charts side by side
- Sentence diff: two columns side by side

Desktop (1024px - 1440px):

Tablet (768px - 1024px):
- Selection panel: two cards still side by side but smaller
- Hero row: three columns compressed
- Distribution charts: stack vertically (A on top, B below)
- Sentence diff: stack vertically (A row then B row for each index)
- Stats table: horizontally scrollable

Mobile (below 768px):
- Selection panel: stack vertically (A card on top, B card below)
- Hero row: stack vertically (A on top, center diff in middle, B below)
- Distribution charts: full width stacked
- Sentence diff: full width stacked per sentence (A then B then divider)
- Stats table: card-based layout instead of table (each row becomes a card with label + A value + B value)
- All charts: full width, reduced height (280px instead of 400px for timeline)
- No horizontal scroll on any section except the stats table on tablet
